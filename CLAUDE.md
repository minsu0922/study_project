# 작업 지침 (사용자 요청 — 반드시 준수)

이 프로젝트는 백엔드 CS 개념을 하나씩 실증하며 배우는 **학습용 포트폴리오**다.
아래 지침은 사용자가 직접 설정한 것으로, 기본 동작보다 우선한다.

## 1. 자율 진행 (영향이 큰 작업만 확인)
- 웬만한 작업은 **물어보지 말고 바로 진행**한다.
- 사용자에게 확인을 받아야 하는 "영향이 큰 작업"의 예:
  - 되돌리기 어려운 것: 파일/데이터 대량 삭제, DB 스키마 파괴적 변경, force push, 외부로 나가는 작업(배포, PR/이슈 생성, 외부 서비스 전송)
  - 프로젝트 방향을 바꾸는 결정: 기술 스택 교체, 아키텍처/설계 계약 변경, MVP 범위 변경
- 그 외 일반 구현·리팩터링·테스트·문서 작성은 확인 없이 진행하고, 결과를 보고한다.

## 2. 상세한 주석
- 작성하는 모든 코드에 **상세한 주석**을 단다.
- 단순히 "무엇을 하는지"가 아니라 **"왜 이렇게 했는지"**(설계 의도·트레이드오프·대안)를 남긴다 — 학습/면접 대본으로 쓰기 위함.
- 클래스/메서드 상단 설명 + 핵심 로직 인라인 주석. 관련 설계 문서(docs/)가 있으면 참조를 적는다.

## 3. 중학생도 이해할 수 있는 설명
- 작업 내용을 보고할 때 **중학생도 이해할 수 있는 눈높이**로 설명한다.
- 전문 용어를 쓰면 곧바로 쉬운 말로 풀어준다 (예: "JWT = 로그인했다는 걸 증명하는 도장이 찍힌 출입증").
- 비유·예시를 적극 활용하고, 왜 필요한지부터 설명한다.
